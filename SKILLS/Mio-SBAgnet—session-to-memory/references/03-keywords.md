# 关键词规范

## 提取规则

每条记忆自动打 3-8 个关键词。提取来源：分类名 + 正文标题 + 事件摘要中的核心名词。

## 要求

- 格式: YAML 列表 `[词1, 词2, 词3]`
- 数量: 3-8 个
- 语言: 中文优先，技术名词保留原文
- 粒度: "搜索这个词能精准定位到这条记忆"——不能太宽（"AI"）也不能太窄（整句话）

## 去口语映射

| 正文中的口语 | 关键词 |
|-------------|--------|
| "那个 hook 没配上" | `hook, 配置缺失` |
| "试了一下发现不行" | `[具体方案名], 排除` |
| "爸爸说以后不要这样" | `偏好, [具体行为]` |
| "查了一下官方文档" | `[文档名], 外部参考` |

## 去重融合规则

### 会话内去重

```
同一 sessionId + 同一分类 + 标题语义相似（LLM 判断）→ 合并为同一条，补充正文细节
同一 sessionId + 同一分类 + 标题相关但不同事件 → 各自保留，双向写 关联: [[...]]
```

### 跨会话级联判断

不再用数值打分。改用两步级联：

```
第一步 — LLM 判断是否为同一事件：是 / 否 / 不确定
  是 → 合并，更新时间线（追加"## YYYY-MM-DD 更新"段）
  否 → 独立新建
  不确定 → 进入第二步
第二步 — 关键词辅助裁决：
  重叠 ≥ 3 → 写入 关联: [[...]]，各自保留
  重叠 < 3 → 独立新建
```

同一事件的标准：同一触发原因 + 同一结论/结果。关键词辅助用于 LLM 不确定时的兜底判断。

## 阈值联动

- 同类**错误**累计 ≥ 3 → 提醒立法程序令 §33
- 同类**模式**累计 ≥ 3 → 提醒（正→偏好固化，负→触发立法）
- **问题**累计 ≥ 5 → 提醒规划优先级审查
- 同关键词组出现 ≥ 5 条记忆 → 建议提升为不变量或创建专门讨论记录

## 示例

一条关于 Harness hook 缺失的记忆：
```yaml
关键词: [Harness, hook, settings.json, SessionStart, 执行层, 宪法第十五条]
```

一条关于 DeepSeek 缓存机制的记忆：
```yaml
关键词: [DeepSeek, 前缀缓存, KV Cache, 递进展开, token优化, 法索引]
```
