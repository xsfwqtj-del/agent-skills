# 关键词规范

## 提取规则

每条记忆自动打 3-8 个关键词。提取来源：分类名 + 正文标题 + 事件摘要中的核心名词。

## 要求

- 格式: YAML 列表 `[词1, 词2, 词3]`
- 数量: 3-8 个
- 语言: 中文优先，技术名词保留原文
- 粒度: "搜索这个词能精准定位到这条记忆"——不能太宽（"AI"）也不能太窄（整句话）

## 去口语映射

| 正文中的口语 | 关键词 |
|-------------|--------|
| "那个 hook 没配上" | `hook, 配置缺失` |
| "试了一下发现不行" | `[具体方案名], 排除` |
| "爸爸说以后不要这样" | `偏好, [具体行为]` |
| "查了一下官方文档" | `[文档名], 外部参考` |

## 去重融合规则

```
新记忆写入前:
  提取关键词 → 与已有记忆比对

  重叠 ≥ 3 个关键词:
    ├── 同事件 + 同分类 → 合并为同一条，补充正文细节
    ├── 同事件 + 不同分类 → 各自保留，双向写 关联: [[...]]
    ├── 不同事件 + 高重叠:
    │     ├── 同类错误 ≥ 3 → 触发"模式"升级提醒 + 立法阈值
    │     ├── 同类偏好 ≥ 3 → 触发偏好固化提醒
    │     └── 其他 → 写入 关联: [[...]]
    └── 不同事件 + 中等重叠(1-2个) → 如有因果关系 → 写关联; 否则各自独立

  重叠 = 0 → 新建独立记忆
```

## 阈值联动

- 同类**错误**累计 ≥ 3 → 提醒立法程序令 §33
- 同类**模式**累计 ≥ 3 → 提醒（正→偏好固化，负→触发立法）
- **问题**累计 ≥ 5 → 提醒规划优先级审查
- 同关键词组出现 ≥ 5 条记忆 → 建议提升为不变量或创建专门讨论记录

## 示例

一条关于 Harness hook 缺失的记忆：
```yaml
关键词: [Harness, hook, settings.json, SessionStart, 执行层, 宪法第十五条]
```

一条关于 DeepSeek 缓存机制的记忆：
```yaml
关键词: [DeepSeek, 前缀缓存, KV Cache, 递进展开, token优化, 法索引]
```
