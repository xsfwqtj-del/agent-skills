# Mio 文档维护自检

> 在回复爸爸之前执行。不靠记忆——每次读这个文件。
> **权威信息来源：`docs/文档地图.md`「每份文档的职责边界与更新触发」表。**

## 规则

**发现漏了的项 → 先补上，再回复。**

**数字不从脑子里拿——从 `docs/STATS.md` 取。** 测试数/模块数以 STATS.md 为唯一来源。

**信息不重复登记——每份文档管什么、不管什么，看文档地图的职责边界表。**

---

## 改了什么 → 更新什么（速查）

> 完整版在 `docs/文档地图.md`「改了什么→更新什么」。这里列最常用的。

```
改了 hermes/plugins/mio/*.py → STATS.md + 架构.md + 状态.md + 待办.md（+ 领域词汇表如有新术语）
改了 docs/                   → 文档地图.md（文件增删时）+ 归档旧文件
改了 .specify/               → 对应 spec/plan + 状态.md + 待办.md
新增设计决策                 → adr/ + 设计.md（如影响设计哲学）
发现偏移                     → .specify/memory/偏移/ + 状态.md（偏移表追加一行）
```

## 检查清单

### 一、改了代码？（看 git diff 有哪些文件）

- [ ] **docs/STATS.md** — 数字变了？**先更新这个**，再同步其他文档
- [ ] **docs/架构.md** — 模块描述/数据流/性能数字？
- [ ] **docs/状态.md** — 偏移数/版本号？
- [ ] **docs/待办.md** — 完成项打勾/新问题写入？
- [ ] **领域词汇表** — 新术语/旧术语废弃？
- [ ] **docs/设计.md** — 设计决策/架构哲学变了？（代码改动很少触发这个）
- [ ] **docs/项目总览.md** — 项目定位变了？（很少变，小改动不触发）
- [ ] **docs/测试指南.md** — 测试策略/分类变了？
- [ ] **CLAUDE.md** — 开发规范/启动协议变了？

### 二、改了文档？

- [ ] **文档地图** — 文件增删/改名/职责变化？更新触发条件还准吗？
- [ ] **归档** — 旧文件移入 `docs/归档/` 并加了时间戳？
- [ ] **服从链** — 下层文档需要同步？

### 三、改了 spec？

- [ ] **spec.md** — 格式完整？（原话 + 原因 + 效果 + 场景 + 边界条件）
- [ ] **docs/待办.md** — 优先级排入了？

### 四、Spec Kit 合规

- [ ] **命令管道** — specify → plan → tasks → analyze → checklist → implement？
- [ ] **CC Plan Mode** — 没误用 EnterPlanMode？
- [ ] **产物落位** — 在 `.specify/memory/` 对应目录？
- [ ] **偏移报告** — 跑过 `/speckit.analyze`？

### 五、通用

- [ ] **偏移** — 新偏移写入 `.specify/memory/偏移/` + `状态.md` 偏移表追加？
- [ ] **宪法合规** — §I 物理隔离 / §III 双线程 / §IV 记忆链路 / §V 八类记忆
