# MioForever 文档整理

## 铁律

1. **规范永远遵循 `docs/00_入口/文档生命周期.md`**。本技能不钉死任何命名规则，每次执行前必须重读生命周期文档获取当前规范。
2. **不改文档内容**。只做文件名、路径引用、元数据更新。不顺手修错字、调整格式、优化表述。
3. **归档文件的旧引用可以保留**。`docs/90_归档/` 中的历史引用代表当时状态，不需要同步为新路径。
4. **重命名是移动，不留副本**。不保留旧名备份，避免双入口。

## 执行流程

### 第 1 步：读取规范

```
Read docs/00_入口/文档生命周期.md
```

确认当前：
- 文件命名格式（字段顺序、分隔符、时间格式）
- 阅读层级定义（00-09 / 10-19 / 20-29 / ...）
- 文档身份词表（状态、计划、映射、PRD、设计、任务书...）
- 例外清单（哪些文件不强制套用命名）
- 重命名规则（何时更新文档地图、何时同步项目状态）

### 第 2 步：列出目标目录

```bash
ls -la "docs/<目标目录>/"
```

记录每个文件的：
- 当前文件名
- 修改时间（用于命名中的时间字段）
- 文件职责（从内容标题或文档地图中确认）

### 第 3 步：制定映射表

根据规范为每个文件生成新名称。输出映射表给用户确认：

| 当前文件名 | 新文件名 | 阅读层级 | 文档身份 | 关联对象 |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |

注意：
- 阅读层级取文件实际职责对应的层级，不是目录编号
- 时间戳取文件最后修改时间（`ls -la` 中的时间）
- 符合例外条件的文件标出"例外：保留原名"
- 关联对象要稳定、短，不写成长句

### 第 4 步：用户确认

展示映射表，确认后再执行。用户说"行""好""执行"才进入下一步。

### 第 5 步：执行重命名

```bash
cd "docs/<目标目录>/"
mv "旧名.md" "新名.md"
```

一次执行所有 rename，不要分批。

### 第 6 步：搜索旧引用

对每个旧文件名，全项目搜索引用：

```
Grep: pattern="旧文件名\.md"
       path=<项目根>
       glob="*.md"
```

记录所有命中文件（排除 `docs/90_归档/` 和 `.git/`）。

### 第 7 步：更新引用

按优先级分批：

1. **文档地图** — `docs/00_入口/文档地图.md`（引用中枢）
2. **目标目录内部** — 重命名文件之间的交叉引用
3. **文档生命周期** — 如果生命周期文档中引用了这些文件
4. **其他活跃 docs/** — 入口、宪法、ADR、指南、日志
5. **外部引用** — `.claude/skills/`、`.claude/agent-knowledge/`、`CLAUDE.md`

对于 `docs/<目标目录>/` 前缀的引用路径：也需替换（路径中的目录部分不变，文件名部分变）。

对于 markdown 链接 `[text](oldname.md)`：同步更新为 `[text](newname.md)`。

对于纯文件名引用（同目录内）：替换为新文件名。

**注意**：`docs/90_归档/` 中的旧引用不更新。`docs/70_开发日志/` 中的引用视情况——如果日志本身是活跃参考则更新，纯历史记录可保留。

### 第 8 步：验证

```bash
# 全量搜索旧文件名（排除归档）
Grep: pattern="旧文件名1\.md|旧文件名2\.md|..."
       glob="*.md"
       path=<项目根>
```

确认：
- 活跃文档中无旧文件名残留
- 归档中的旧引用是有意保留
- 新文件名指向的文件确实存在

### 第 9 步：落盘

1. **更新 `docs/00_入口/文档地图.md`** — 同步所有引用路径
2. **更新 `docs/20_当前阶段/项目状态.md`**（或当前的状态文件） — 追加"最近一次更新"条目记录此次变更
3. **告知用户** — 改了什么、改了多少处、哪些文件受影响、有无风险

## 不做什么

- 不创建新文档（那是 doc-guardian 的事）
- 不修改文档内容（只改引用路径）
- 不归档文档（那是 doc-guardian 的事）
- 不定制命名规范（规范在 文档生命周期.md 里）
- 不更新 Tolaria 知识库（那是会话收尾的事）
- 不 git commit（等用户指令）

## 与其他技能的关系

| 场景 | 用哪个技能 |
|---|---|
| 判断文档该放哪个目录、何时归档 | `mio-doc-guardian` |
| 执行文件重命名和引用同步 | `mio-doc-organizer`（本技能） |
| 会话开始/结束的状态管理 | `mio-session-orchestrator` |
