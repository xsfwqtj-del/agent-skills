# 讨论蒸馏法

> 版本: v1.0.1 | 更新: 补 trigger:auto + CSO 修复 + 常见错误 + Red Flags + 不适用场景

> 不是总结——是蒸馏。保留每一个推导跳步的原文证据，确保失忆后能无歧义重建整个推理过程。

---

## 触发判断

| 讨论特征 | 要不要蒸馏 |
|----------|-----------|
| 跨越 3+ 个话题领域 | ✅ 蒸馏 |
| 有反问、纠正、推翻原方案 | ✅ 蒸馏 |
| 持续超过 30 轮对话 | ✅ 蒸馏 |
| 简单问答、查资料 | ❌ 不需要 |
| 单文件小改动讨论 | ❌ 不需要 |

---

## 四步流程

```
1. 逆时针回溯
   从最后一个结论开始，往前追问"这个结论是怎么来的？"
   一直追到第一个触发事件
      ↓
2. 分幕分组
   把相关链条归入同一幕。一幕 = 一个阶段性的核心问题
   幕与幕之间是"老问题被推翻 → 新问题浮现"的关系
      ↓
3. 引用锚定
   每个关键推理步必须附原始引用（爸爸的原话、搜索结论的原文）
   不凭记忆归纳——翻对话记录，找原文
      ↓
4. 产物索引
   每个链条标注产出了什么文件、方案、代码提交
   确保"讨论 → 产物"的因果关系完整
```

## 输出格式

```markdown
# [主题] — 完整链路追踪

> 日期 · 从"触发"到"结论"的完整推导链
> 原文 + 归纳 + 决策点，确保零遗漏

---

## 第X幕：[阶段性主题]

### 链条 N：[这一个推理环节的标题]

**原文**：
> "爸爸的原话引用"

**背景**：（如果需要，补充讨论时已有的上下文）

**关键发现**：（一句话 —— 这个环节导致了什么认知变化）

**决策记录**：（如果产出了方案/代码/文档）

---

## 全链路缩略

（一棵向右的树，每个 → 是一个链条）

---

## 附录：关键文件索引

| 文件 | 说明 |
|------|------|
| ... | ... |
```

---

## 质量验收标准

蒸馏完成后自检：

- [ ] 每条"发现"能找到对应的原文引用
- [ ] 每个文件/方案能找到对应的讨论链条
- [ ] 全链路缩略图可以 30 秒扫完主线
- [ ] 一个没参与讨论的人读完能复述"我们为什么做了这些决定"
- [ ] 没有"后来就..."的模糊跳步 — 每一步的推理依据都写明了
- [ ] 归档于 `docs/plans/`（讨论产出讨论过程文档）

## 常见错误

| 错误 | 预防 |
|------|------|
| 凭记忆归纳而非翻原文 | "后来就..."式跳步是红线。每一步必须有引用 |
| 只写结论不追溯推导过程 | 从结论往前问"这个怎么来的"，一直追到触发事件 |
| 把蒸馏当总结——省略推理链只给要点 | 蒸馏保留跳步，总结压缩信息。二者不同 |
| 跳过了文件索引 | 讨论→产物的因果关系是完整链路的一部分 |

---

## Red Flags

- "这个讨论很简单不需要蒸馏"
- "大概就是这样的"
- "我记不太清但应该..."
- "后来就做了..."

**以上想法出现时，回到原文翻对话记录。**

---

## 不适用场景

- 简单问答、单文件小改动 → 无需蒸馏
- 批量 JSONL 会话提取 → 用 Mio-SBAgnet—session-to-memory
- 日常闲聊无决策产出 → 无需蒸馏
