# Mio-SBAgnet—技能汉化

> 版本: v0.1.0 | 更新: 初始版本

---

## 概述

将本地已安装的英文技能汉化。不做远程抓取，不覆盖原文件。核心原则：**翻译前先汇报确认、翻译后再次确认**。

---

## 工作流程

```
1. 确认目标      — 用户指定要汉化的技能目录路径
       ↓
2. 阅读全部      — 读取 SKILL.md + 所有子文件，理解技能功能和目录结构
       ↓
3. 第一轮确认    — 汇报技能概述 + 文件处理清单 + 提议的中文触发词 + 翻译深度
                   用户确认或修改。触发词不符按用户说的改。
       ↓
4. 执行          — 复制整个目录到 -zh → 选择性翻译 → 删除 CREATION-LOG.md
       ↓
5. 第二轮确认    — 汇报翻译结果（触发词、翻译了哪些文件），用户确认或修改
       ↓
6. 输出就位      — -zh 目录即最终版本
```

---

## 每步具体要求

### 1. 确认目标

用户指定技能目录路径。路径可以是绝对路径或相对于当前技能目录的路径。如果用户只说技能名（如"汉化 brainstorming"），先在当前技能目录和 `~/.claude/skills/` 下搜索匹配。

### 2. 阅读全部

读取目标目录下的**全部文件**，不止 SKILL.md。建立文件清单，标注每类文件：

| 文件类型 | 判断依据 | 处理方式 |
|:---|:---|:---|
| 主文件 | `SKILL.md` | 必须翻译 |
| 引用子文档 | `.md` 文件，被 SKILL.md 链接引用 | 深度时翻译，轻量时复制原文 |
| 独立 md | `.md` 文件，未被 SKILL.md 引用（如 CREATION-LOG.md） | CREATION-LOG 不复制，其余视内容决定 |
| 代码脚本 | `.sh` `.js` `.ts` `.py` | 原样复制 |
| 示例/测试 | `test-*.md` `example*` `*-example.*` | 原样复制 |
| 配置/模板 | `assets/` `references/` 中的非 md 文件 | 原样复制 |

### 3. 第一轮确认（核心步骤）

向用户呈现以下信息，等确认后再动手：

```
## [技能原名]

**这个技能是干什么的**
[用中文概述，2-3 句话]

**目录结构**
[文件清单，标注哪些会翻译、哪些原样复制]

**翻译深度**: [轻量汉化 / 深度汉化]
理由: [为什么]

**建议的中文触发词**
- 搜索关键词: [用户会用哪些中文词搜到这个技能]
- description: [中文 description，让用户判断是否准确]
```

用户可能回复：
- 批准 → 进入步骤 4
- 修改触发词 → 按用户说的调整后再次确认
- 修改深度 → 按用户说的调整后再次确认
- 取消 → 不执行

### 4. 执行翻译

按确认后的深度执行。具体翻译规则见下方「翻译原则」。

操作顺序：
1. 复制整个目录到 `{原目录名}-zh`
2. 删除 CREATION-LOG.md（如存在）
3. 翻译需要翻译的 .md 文件
4. 更新 Markdown 内部链接（被翻译的文件名需对应更新）

### 5. 第二轮确认

翻译完成后汇报：

```
## 翻译完成

**最终中文触发词**: [和用户确认过的触发词]
**翻译深度**: [轻量/深度]
**已翻译文件**: [列出]
**原样保留文件**: [列出]
**输出路径**: `{原目录}-zh/`
```

用户确认无误则完成。有修改意见则回到步骤 4 调整。

### 6. 输出就位

- 原目录无任何修改
- `-zh` 目录为独立可用的完整技能
- 告知用户："可以用 `{中文触发词}` 搜索到这个技能了"

---

## 翻译深度判断

Agent 读完全部文件后自主判断，不预定义规则。判断逻辑：

**轻量汉化**（只译 frontmatter 触发层）：
- name 加 `-zh` 后缀
- description 翻译为中文
- 正文和子文档原样保留
- 适用于：内容以代码/命令为主、面向有英文基础的开发者、子文档简短且非自然语言密集

**深度汉化**（全文翻译）：
- 轻量汉化的全部内容
- SKILL.md 正文全译
- 被引用的子文档（`.md`）全译
- 适用于：内容以概念/流程/规则为主、有大量英文自然语言叙述、面向中文母语用户

---

## 翻译原则

- **YAML frontmatter**: name 加 `-zh` 后缀，description 翻译为中文触发条件
- **不翻译**: 代码块、命令行、文件路径、API 名称、行内代码、流程图、ASCII art
- **专业术语**: 首次出现保留英文原文 + 中文注释，如 `Harness（外骨骼/编排层）`
- **Markdown 结构**: 完全保留，不增删章节
- **Markdown 内部链接**: 如果引用的 .md 文件被翻译了，更新链接指向翻译后的文件名
- **测试用例/示例**: 不翻译，原样保留
- **CREATION-LOG.md**: 不复制
- **代码脚本**: 不翻译，原样复制

---

## 常见情况

- `-zh` 目录已存在 → 提示用户，询问覆盖还是跳过
- 技能本身就是中文 → 告知用户"这个技能已经是中文的，无需汉化"
- 技能只有 SKILL.md 无子文件 → 简化文件清单，跳过子文档判断
